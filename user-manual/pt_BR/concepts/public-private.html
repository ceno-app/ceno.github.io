Navegação pública versus privativa

Because of the many techniques used to overcome connectivity issues, CENO may become a convenient way for you to get all kinds of Web content. And, as you may have already read in previous sections, whenever you retrieve and seed a page using CENO Browser, it becomes available to others. There may be some content, however, that you do not wish to share (or you do not want to let others know that you are trying to or did retrieve), and fortunately CENO can help you in this instance as well.

The default mode when you launch the application is public browsing. In it, CENO accesses Web content as described previously:

  1. Tenta-se o acesso direto.
  2. Em caso de falha, procura-se o cache distribuído.
  3. Em caso de falha, o conteúdo é requisitado através de um injetor (talvez via outro cliente).

CENO also has a private browsing mode. In it, the distributed cache is never searched, and injection is never attempted:

  1. Tenta-se o acesso direto.
  2. Em caso de falha, um injetor é contatado (talvez via outro cliente) e usado como um servidor proxy normal. Note que, nesse caso, nem o injetor nem o seu cliente atualizam o cache distribuído com a sua página.

O comportamento diferente resulta em características diferentes. Assim, no modo público:

  1. Você tem mais chances de obter conteúdo Web e ajudar outras pessoas a obter conteúdo (de você).
  2. Páginas com conteúdo dinâmico (exemplo, atualizadas em tempo real) podem quebrar de maneiras óbvias ou sutis.
  3. Páginas que requerem autenticação não funcionam (pois senhas e cookies são removidos pelo cliente).
  4. Algumas atividades de navegação podem ser vazadas para outros usuários (veja os riscos).
  5. Algumas atividades de navegação podem ser vazadas para injetores (veja os riscos).
  6. Você precisa confiar nos injetores para recuperar e assinar conteúdo Web.

Enquanto estiver no modo privativo:

  1. You may not be able to access blocked Web content if international connectivity is too scarce; even if you could, other CENO users would not get that content from you.
  2. As páginas com conteúdo dinâmico provavelmente funcionarão.
  3. Páginas que requerem autenticação podem funcionar (quando sua conexão está protegida por HTTPS, o injetor não vê suas senhas).
  4. A atividade de navegação não é vazada para outros usuários.
  5. Uma quantidade limitada de atividade de navegação é vazada para injetores (com HTTPS, apenas o nome ou endereço do servidor de origem).
  6. Você não precisa confiar nos injetores (com HTTPS, a segurança usual baseada em certificados ainda funciona).

In conclusion: if you are using CENO to read the news, watch videos, browse Wikipedia and other static resources that are otherwise censored in your network, consider using the default public browsing mode. And if you want to login to Twitter or edit your WordPress website, use private browsing mode.

Por favor, leia a seção sobre riscos para uma explicação mais detalhada. Note também que o seu cliente pode continuar a operar como uma ponte e um semeador independentemente da navegação pública ou privativa. Explicamos isso em detalhes na seção Ajudando outras pessoas do manual.
