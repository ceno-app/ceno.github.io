Annexe : L'interface du client Ouinet

Le client Ouinet (tel qu'exécuté par exemple par le navigateur CENO) propose une interface contenant des informations et des actions qui peuvent s'avérer utiles au débogage du client. La plupart d'entre elles sont également accessibles via l'extension de CENO, dans la page Paramètres de CENO.

L'interface est accessible par le biais de n'importe quel navigateur web fonctionnant sur le même appareil (vous pouvez également utiliser CENO). Son adresse par défaut est http://127.0.0.1:8078/. Voici un aperçu de l'interface lorsque vous l'ouvrez.

Les éléments affichés dans la page incluent :

  • Un lien permettant d'activer le client en tant qu'autorité de certification (CA) dans votre navigateur, car le client doit intercepter du trafic HTTPS.Vous n'en aurez besoin que lors de l'utilisation d'un navigateur classique pour tester le client Ouinet, auquel cas vous devrez également configurer ses proxy HTTP/HTTPS sur 127.0.0.1:8077, et activer manuellement l'Extension CENO pour que l'injection fonctionne. Nous vous recommandons vivement d'utiliser un profil de navigateur spécifique, distinct pour cela.

Veuillez noter que tout cela n'est pas nécessaire s'agissant du navigateur CENO, car il est déjà configuré de cette façon.

  • Des boutons pour activer ou non les différents mécanismes utilisés par le client pour accéder au contenu.

    • Une liste déroulante proposant différents niveaux de détail du journal, comme INFO (messages d'information, avertissements et erreurs) ou DEBUG (sortie verbeuse, utile pour signaler des erreurs). Le fichier journal peut également être activé et récupéré à partir d'ici.Lorsque vous activez le fichier journal, son niveau de détail est automatiquement fixé sur DEBUG (mais vous pouvez le modifier selon vos besoins). Et lorsque vous le désactivez, son niveau original est restauré.

  • Informations sur l'état et la version du client. Utiles pour signaler des erreurs.

  • Informations à propos de la connectivité du client et l'adressage des injecteurs. La méthode bep5 par défaut recherche les adresses Internet dans un essaim d'injecteurs BitTorrent, tel qu'expliqué ici.

  • La clé publique, utilisée pour vérifier les signatures des injecteurs dans le cache distribué.

  • Informations concernant votre cache local, comme l'âge maximal du contenu, la taille approximative du cache, un bouton pour le purger complètement et un lien vers la liste des entrées de cache annoncées.

  • Le répertoire du cache externe, statique lorsqu'il est activé (CENO ne l'utilise pas actuellement).
