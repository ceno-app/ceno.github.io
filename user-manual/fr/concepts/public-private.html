Censorship.no! Manuel de l'Utilisateur

Navigation publique/privée

Grâce aux nombreuses techniques utilisées pour surmonter les problèmes de connectivité, CENO peut devenir très pratique pour obtenir toutes sortes de contenus web. Et, comme vous l'avez sans doute lu dans les sections précédentes, chaque fois que vous récupérez et diffusez une page web par le biais du navigateur CENO, elle devient disponible pour les autres. Il se peut toutefois que vous ne souhaitiez pas partager certains contenus (ou que d'autres sachent que vous essayiez de les récupérer ou que vous les ayez déjà récupérés). Heureusement, CENO vous apporte une solution à cela.

Lorsque vous lancez l'application, le mode par défaut est navigation publique. Ce mode permet à CENO d'accéder au contenu web de la façon suivante :

  1. Tentative d'accès direct.
  2. À défaut, une recherche est effectuée dans le cache distribué.
  3. À défaut, le contenu est demandé via un injecteur (éventuellement via un autre client).

CENO dispose également d'un mode de navigation privée. Dans ce mode, le cache distribué et l'injection ne sont jamais sollicités :

  1. Tentative d'accès direct.
  2. À défaut, un injecteur est contacté (éventuellement via un autre client) et utilisé comme un serveur proxy normal. Notez que dans ce cas, ni l'injecteur ni votre client ne mettent à jour le cache distribué avec votre page.

Cette différence de comportement est liée à certaines caractéristiques. Ainsi, en mode public :

  1. Vous aurez de meilleures chances d'obtenir du contenu web et d'aider les autres à recevoir ce contenu également (de votre part).
  2. Les pages dont le contenu est dynamique (mis à jour en temps réel) peuvent être cassées de façon évidente ou subtile.
  3. Les pages nécessitant une authentification ne fonctionnent pas (car les mots de passe et les cookies sont supprimés par le client).
  4. Certaines activités de navigation peuvent être divulguées à d'autres utilisateurs (cf. risques).
  5. Certaines activités de navigation peuvent être divulguées aux injecteurs (cf. risques).
  6. Vous devez faire confiance aux injecteurs pour récupérer et signer du contenu web.

Alors qu'en mode privé :

  1. Il se peut que vous ne puissiez pas accéder à du contenu web bloqué si la connectivité internationale est trop faible ; et même si c'était le cas, les autres utilisateurs de CENO ne pourraient pas obtenir ce contenu.
  2. Les pages dont le contenu est dynamique fonctionneront probablement.
  3. Les pages nécessitant une authentification peuvent fonctionner (lorsque votre connexion est protégée par HTTPS, l'injecteur ne voit pas vos mots de passe).
  4. L'activité de navigation n'est pas divulguée aux autres utilisateurs.
  5. Une quantité limitée d'activité de navigation est divulguée aux injecteurs (seulement le nom ou l'adresse du serveur d'origine, avec le protocole HTTPS).
  6. Vous n’avez pas besoin de faire confiance aux injecteurs (avec HTTPS, la sécurité habituelle basée sur les certificats fonctionne).

En conclusion : si vous utilisez CENO pour consulter des actualités, regarder des vidéos, naviguer sur Wikipedia et autres ressources statiques, qui seraient à défaut censurées dans votre réseau, envisagez plutôt d'utiliser le mode de navigation publique. À l'inverse, si vous souhaitez vous connecter à Twitter ou éditer votre site Internet sous WordPress, utilisez le mode navigation privée.

Veuillez vous référer à la section sur les [risques] (risks.md) pour une explication plus détaillée. Notez également que votre client peut continuer à fonctionner comme passerelle et diffuseur, indépendamment de la navigation publique ou privée. Nous expliquons ce fonctionnement plus en détail dans la section Aider les autres du manuel.
