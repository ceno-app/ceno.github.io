Censorship.no! Manuel de l'Utilisateur

La navigation publique comparée à la navigation privée

Grâce aux nombreuses techniques utilisées pour surmonter les problèmes de connectivité, Ceno pourrait devenir pour vous un moyen pratique d’obtenir toutes sortes de contenus Web. Et, comme vous l’avez peut-être déjà lu dans les sections précédentes, chaque fois que vous récupérez une page à l’aide du Navigateur Ceno et que vous la propagez, elle devient accessible à d’autres personnes. Cependant, vous ne souhaiterez peut-être pas partager certains contenus (ou vous ne voudrez pas que d’autres personnes sachent que vous essayez de les récupérer ou que vous les avez récupérés), et heureusement, Ceno peut aussi vous aider dans ce cas.

Quand vous lancez l’application, le mode par défaut est navigation publique. Dans ce mode, Ceno accède aux contenus Web comme décrit précédemment :

  1. Un accès direct est tenté.
  2. En cas d’échec, le cache distribué est consulté.
  3. En cas d’échec, le contenu est demandé par l’intermédiaire d’un injecteur (peut-être par l’intermédiaire d’un autre client).

Ceno propose aussi d’un mode de navigation privée. Dans ce mode, le cache distribué n’est jamais consulté, et aucune injection n’est jamais tentée :

  1. Un accès direct est tenté.
  2. En cas d’échec, un injecteur est contacté (peut-être par l’intermédiaire d’un autre client) et utilisé comme un serveur mandataire normal. Notez que dans ce cas ni l’injecteur ni votre client ne mettent à jour le cache distribué avec votre page.

Ce comportement différent entraîne des caractéristiques différentes. Ainsi, en mode public :

  1. Vous avez plus de chances d’obtenir du contenu Web et d’aider les autres à obtenir ce contenu (de vous).
  2. Les pages dont le contenu est dynamique (p. ex. mis à jour en temps réel) peuvent se briser de manière évidente ou subtile.
  3. Les pages qui nécessitent une authentification ne fonctionnent pas (car les mots de passe et les témoins sont supprimés par le client).
  4. Certaines activités de navigation pourraient être divulguées à d’autres utilisateurs (voir risques).
  5. Certaines activités de navigation pourraient être divulguées aux injecteurs (voir risques).
  6. Vous devez faire confiance aux injecteurs pour récupérer et signer le contenu Web.

Alors qu’en mode de navigation privée :

  1. Il se peut que vous ne puissiez pas accéder au contenu Web bloqué si la connectivité internationale est trop faible ; même si vous y parveniez, les autres utilisateurs de Ceno ne pourraient pas obtenir ce contenu de vous.
  2. Les pages dont le contenu est dynamique fonctionneront probablement.
  3. Les pages qui nécessitent une authentification pourraient fonctionner (si votre connexion est protégée par HTTPS, l’injecteur ne voit pas vos mots de passe).
  4. L’activité de navigation n’est pas divulguée aux autres utilisateurs.
  5. Une activité de navigation limitée est divulguée aux injecteurs (avec HTTPS, le nom ou l’adresse du serveur d’origine seulement).
  6. Vous n’avez pas à faire confiance aux injecteurs (avec HTTPS, la sécurité habituelle fondée sur les certificats fonctionne encore).

Pour conclure : si vous utilisez Ceno pour lire les nouvelles, regarder des vidéos, parcourir Wikipédia et d’autres ressources statiques qui sont autrement censurées sur votre réseau, envisagez d’utiliser le mode navigation privée par défaut. Et si vous voulez vous connecter à Twitter ou modifier votre site Web WordPress, utilisez le mode navigation privée.

Pour plus de précisions, consultez la section sur les risques. Notez aussi que votre client peut continuer à fonctionner en tant que pont et propagateur en mode de navigation public ou privé. Nous couvrons ce point en détail dans la section Aider les autres du manuel.
