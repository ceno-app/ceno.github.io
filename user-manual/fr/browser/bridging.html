Censorship.no! Manuel de l'Utilisateur

Aider les autres utilisateurs de CENO à naviguer sur le web

Un réseau pair-à-pair se construit à partir de chaque nœud qui y est connecté (oui, y compris vous !). Plus il y a de nœuds, plus le réseau se renforce et devient polyvalent. Si vous utilisez le navigateur CENO dans un pays qui ne pratique pas la censure sur Internet (ou dans une moindre mesure que certains), vous pouvez aider les autres utilisateurs de CENO en devenant un nœud passerelle. Vous commencerez alors à acheminer le trafic entre les clients vivant dans des pays fortement censurés et les injecteurs de CENO. Vous ne serez pas en mesure de voir leur trafic (qui sera transmis à travers un tunnel crypté), et aucune trace de ce trafic ne restera dans votre appareil.

Remarque : La configuration décrite dans cette section peut aussi aider votre appareil à transmettre efficacement du contenu aux autres sur le cache distribué. Pensez donc à l'appliquer lorsque vous utilisez CENO dans un pays qui pratique la censure (mais gardez à l'esprit les risques liés à la diffusion de ce contenu à d'autres personnes).

Comment devenir une passerelle CENO

Cette fonctionnalité est déjà intégrée au navigateur CENO. Votre appareil devra être connecté à un réseau Wi-Fi dont l'UPnP est activé ou dont la redirection de port explicite est configurée pour CENO. Voir les sections suivantes pour plus de détails.

Toutefois, veuillez noter qu'Android n'autorisera votre appareil mobile à faire office de passerelle que lorsque vous l'utilisez activement. À défaut, les fonctions d'économie d'énergie limiteront le fonctionnement de CENO.

Note technique : Ceci est principalement lié au [mode Doze](https://developer.android.com/training/monitoring-device-state/doze-standby « Android Developers - Optimize for Doze and App Standby ») d'Android, qui ralentit le fonctionnement de la bibliothèque native Ouinet. Malheureusement, la désactivation de l'optimisation de la batterie pour CENO ne semble pas agir sur Ouinet. Votre appareil peut également inclure ses propres fonctions d'économie d'énergie qui peuvent interférer avec CENO ; veuillez consulter les informations de [Don't kill my app!][Don't kill my app!] concernant la marque de votre appareil.

Ainsi, si vous souhaitez que CENO agisse comme une passerelle permanente, toujours joignable, en plus d'un réseau Wi-Fi correctement configuré, il vous sera nécessaire de :

  1. Gardez votre appareil constamment branché sur le secteur.

    1. Gardez l'écran de l'appareil allumé en permanence.Un moyen pratique d'y parvenir sans consommer beaucoup d'énergie et sans éclairage permanent est d'utiliser l'économiseur d'écran d'Android : activez-le dans Paramètres / Affichage / Économiseur d'écran (ou Daydream dans certaines versions), choisissez le widget Horloge, sélectionnez Quand activer l'économiseur d'écran dans le menu, puis En charge ou Dans les deux cas. Une horloge estompée apparaitra sur fond noir pour indiquer que l'appareil est inactif.

    Veuillez noter que vous ne devez pas utiliser le bouton d'alimentation pour verrouiller l'appareil, car cela éteindrait l'écran. Attendez plutôt que l'appareil se verrouille de lui-même avec l'écran allumé.

Si cette configuration n'est pas envisageable dans votre cas, ne vous découragez pas pour autant ! Si vous avez un ordinateur avec une bonne connectivité, qui reste allumé la plupart du temps, veuillez continuez la lecture de cette section.

Mettre en place une passerelle sur un ordinateur

Si votre ordinateur prend en charge les [conteneurs Docker][docker], vous pouvez faire fonctionner un client CENO préconfiguré qui fera office de passerelle. Si Docker n'est pas encore installé, veuillez suivre les instructions pour [installer le moteur Docker][docker-install] sur votre plateforme. Pour les dérivés de Debian, comme Ubuntu ou Linux Mint, vous pouvez simplement exécuter : sudo apt install docker.io

Pour lancer un conteneur client CENO, il vous suffit d'exécuter la commande suivante dans un terminal (elle a l'air effrayante, mais vous pouvez simplement la copier et la coller) :

sudo docker run --name ceno-client \
  -dv ceno:/var/opt/ouinet --network host \
  --restart unless-stopped equalitie/ceno-client

Si votre ordinateur n'est pas basé sur GNU/Linux, la commande sera légèrement différente :

sudo docker run --name ceno-client \
  -dv ceno:/var/opt/ouinet \
  -p 127.0.0.1:8077-8078:8077-8078 -p 28729:28729/udp \
  --restart unless-stopped equalitie/ceno-client

La commande lancera un conteneur nommé ceno-client qui s'exécutera à chaque démarrage, à moins que vous lui demandiez expressément de s'arrêter. Veuillez consulter la [documentation Docker du client CENO][ceno-client-doc] pour plus d'informations sur la façon de manipuler le conteneur.

Note : Ce client n'a pas de Paramètres CENO : lorsqu'il vous est demandé plus bas d'accéder à cette page, ouvrez plutôt le front-end du client, qui contient des informations équivalentes.

Activer l'UPnP sur votre routeur Wi-Fi

[UPnP][UPnP] est le moyen le plus simple de rendre votre navigateur CENO (ou le client de votre ordinateur) accessible au réseau CENO. La page Paramètres de CENO vous indiquera l'état de l'UPnP sur votre réseau local.

Remarque : L'activation d'UPnP sur le routeur Wi-Fi peut exposer les périphériques de votre réseau à des interférences externes. Veuillez prendre [conscience des risques][upnp-risks] et envisager l'utilisation de méthodes alternatives comme expliqué ci-dessous.

Un état comme celui ci-dessous indique que l'UPnP n'est pas disponible ou ne fonctionne pas sur votre routeur WiFi :

État d'accessibilité

undecided

État de l'UPnP

disabled / inactive

L'état ci-dessous indique que l'UPnP fonctionne probablement et que CENO est en train de vérifier la connectivité :

État d'accessibilité

undecided

État de l'UPnP

enabled

L'état ci-dessous indique que l'UPnP fonctionne et que vous pouvez établir des connexions pour d'autres utilisateurs de CENO :

État d'accessibilité

likely reachable / reachable

État de l'UPnP

enabled

Remarque : Même si l'UPnP fonctionne, votre routeur peut ne pas être joignable de l'extérieur. Cela peut être le cas lorsque les Paramètres de CENO signalent des Point limite UDP externes similaires aux adresses [CGNAT][CGNAT] 100.X.Y.Z:N avec X compris entre 64 et 127 (de plus en plus fréquent chez les FAI domestiques), ou aux adresses privées 10.X.Y.Z:N, 172.X.Y.Z:N avec X compris entre 16 et 31, et 192.168.X.Y:N. Si c'est le cas, veuillez contacter votre FAI ou votre administrateur réseau pour obtenir une adresse publique sur votre routeur ou pour créer des redirections de port vers le point limite externe.

Il existe de nombreux routeurs Wi-Fi sur le marché et chacun possède ses propres caractéristiques. Voici une liste des consignes de certains fabricants pour activer l'UPnP :

  • Linksys
  • D-Link
  • Huawei
  • Xfinity
  • TP-Link
  • Netgear
  • Ubiquiti

Utiliser la redirection de port comme alternative à l'UPnP

Au lieu d'activer l'UPnP sur votre routeur, vous pouvez créer une règle de redirection de port pour vous assurer que les connexions du réseau CENO sont bien transférées vers votre appareil. Vous devrez vous connecter à l’interface d'administration du routeur et trouver l’option redirection de port. Pour voir vers quelle adresse IP vous devez rediriger les connexions et le port correspondant, ouvrez la page Paramètres de CENO et regardez dans les Points limites UDP locaux.

Points limite UDP locaux

192.168.1.132:28729

La redirection de port doit viser le protocole UDP (et non TCP). CENO choisit un port aléatoire lors de sa première exécution et le conserve pour les exécutions suivantes, mais l'adresse IP du réseau local de votre appareil peut changer de temps en temps. Vous devez donc consulter régulièrement la page des Paramètres de CENO pour vérifier que votre appareil est accessible au réseau CENO.

Note technique : Vous pouvez également vous assurer que le routeur attribue toujours la même adresse IP à votre appareil (par exemple via un abonnement DHCP statique pour l'adresse MAC de l'appareil).
