Censorship.no! Manuel de l'Utilisateur

Paramètres de CENO

Le navigateur CENO vous permet de modifier des paramètres spécifiques à Ouinet et d’obtenir facilement des informations sur votre client. Cela ne devrait pas être nécessaire pour un fonctionnement normal, mais il peut devenir très utile pour tester différentes stratégies afin de contourner les interférences réseau, ainsi que signaler des problèmes avec l’application.

Note technique : Ces options sont fournies par l'Extension CENO, une extension Firefox installée d'office avec CENO, qui gère l’intégration avec Ouinet. Elle permet ainsi d'injecter du contenu et de récupérer la mémoire-cache de navigation publique, indiquant à l’utilisateur la source du contenu visualisé et l’informant au sujet de nouvelles versions de Ouinet.

Ces fonctionnalités sont disponibles sur une page accessible en sélectionnant CENO dans le menu principal de l’application. Veuillez noter que le menu peut mettre quelques secondes à apparaitre au moment du démarrage de l’application. La page devrait ressembler à cela :

Choix des mécanismes d’accès

Les quatre cases à cocher en haut de la page permettent d’activer ou de désactiver les différents mécanismes ou sources que CENO utilise, en tant que client Ouinet, pour récupérer du contenu en utilisant les onglets de navigation publique ou privée. Toutes les cases sont cochées par défaut.

  • En direct du web (ou accès d'origine) permet à CENO d’essayer d’atteindre le serveur d’origine directement, avant d'essayer les autres mécanismes activés.Bien que ce mécanisme fonctionne aussi bien en mode de navigation privée que publique, le contenu ainsi récupéré ne peut être partagé avec les autres.

Si l'accès à la majorité des contenus web n’est pas particulièrement lent ou coûteux, ce mécanisme est amplement suffisant dans la plupart des cas. Toutefois, ces connexions directes sont susceptibles d'être traquées par votre FAI ou votre gouvernement. Dans une certaine mesure, la désactivation de cette option permet éviter de telles connexions et un suivi trivial (mais pas complètement, cf. [risques](.. /concepts/risks.md)).

De plus, lors de l'accès à un site web via un protocole HTTP non sécurisé (au lieu du protocole HTTPS, davantage sécurisé), un censeur peut intercepter la connexion et fournir à l'utilisateur un faux site, une falsification que CENO ne peut détecter seul. Dans ce cas, il peut être utile de désactiver cette option et d'avoir recours aux autres mécanismes de CENO, plus sûrs. Veuillez consulter la section dépannage pour en savoir plus sur le sujet.

  • Via le réseau CENO (privé) (ou accès proxy) permet à CENO d'utiliser les injecteurs comme des serveurs proxy HTTP normaux afin d'atteindre les serveurs d'origine.Ce mécanisme ne fonctionne qu'en mode de navigation privée.

Lors de l'accès à du contenu par HTTPS, seuls les serveurs d'origine sont en mesure de décrypter le trafic. En utilisant le protocole HTTP simple, l'injecteur peut également voir le trafic non crypté (mais il ne peut tout de même pas signer ou partager son contenu avec les autres). Les autres participants, comme les passerelles, ne verront jamais le trafic non crypté.

  • Via le réseau CENO (public) (ou accès injecteur) permet à CENO de retirer toute information privée des demandes et de les envoyer à un injecteur. L'injecteur reçoit le contenu du serveur d'origine, le signe et le renvoie à CENO, qui va alors le diffuser.Les autres participants (comme les passerelles) ne verront pas le trafic non crypté.

Ce mécanisme ne fonctionne qu'en mode de navigation publique.

  • Partagé par d'autres utilisateurs de CENO permet à CENO d'essayer de récupérer le contenu du cache décentralisé, autrement dit depuis d'autres clients CENO et Ouinet qui le diffusent.Ce mécanisme ne fonctionne qu'en mode de navigation publique.

À propos de votre application

Cette page vous fournit également des informations sur votre application CENO et votre client Ouinet :

  • Taille du cache local indique approximativement de la quantité de stockage utilisée par le contenu diffusé à partir de la mémoire cache locale de votre appareil.
  • Contenu partagé par vous vous permet de vérifier le contenu présent dans votre appareil.
  • État du client Ouinet, si démarré, cela signifie que votre client Ouinet a pu s'exécuter avec succès. À défaut, il existe sans doute des problèmes de connectivité ou une erreur interne. Merci de le préciser dans vos rapports d'erreurs.
  • État d'accessibilité indique si votre appareil est apte à diffuser efficacement du contenu vers les autres clients. À inclure également dans les rapports.
  • État du UPnP indique si CENO a pu demander à votre routeur ou point d'accès d'autoriser les connexions entrantes vers celui-ci. À inclure également dans les rapports.
  • Les Points limite UDP locaux sont les adresses Internet de votre appareil utilisées par CENO pour diffuser du contenu signé vers les autres clients. Elles sont affichées pour permettre de tester et déboguer l'application, et ne doivent en aucun cas être divulguées.
  • Les Points limite UDP externes sont les adresses Internet de votre routeur attribuées au trafic de CENO. Uniquement disponibles avec les routeurs compatibles UPnP, elles sont également utiles pour les diagnostics, et ne doivent pas être divulguées.
  • Les Points limite UDP publics sont les adresses Internet que les clients Ouinet hors de votre réseau voient lorsqu'ils communiquent avec votre appareil. Ils sont également utilisés pour les diagnostics et ne doivent pas être divulgués.
  • Les amorces BitTorrent supplémentaires sont des serveurs utilisés pour aider votre appareil à accéder au réseau BitTorrent, si les serveurs par défaut ne fonctionnent pas. À inclure également dans les rapports. Vous pouvez modifier la liste des hôtes séparés par des espaces (avec des ports facultatifs) pour définir ou ajouter le vôtre, puis choisissez Enregistrer. Les changements seront appliqués lors du prochain démarrage de votre client Ouinet.
  • Navigateur CENO indique la version exacte de CENO que vous utilisez. À inclure également dans les rapports.
  • Extension CENO montre la version de l'extension qui associe Firefox à CENO. À inclure également dans les rapports.
  • Ouinet montre la version de Ouinet qui accompagne CENO. À inclure également dans les rapports.
  • Protocole Ouinet est le numéro de version du protocole que CENO utilise pour communiquer avec les autres clients et injecteurs Ouinet. À inclure également dans les rapports.

Purge du cache local

À côté de la valeur Taille du cache local précitée, vous trouverez un bouton permettant d'arrêter la diffusion et de déposer tout le contenu partagé par votre appareil dans Ouinet. Cela vous permet de libérer de l'espace de stockage dans votre appareil tout en gardant les autres paramètres CENO en favoris.

Si vous souhaitez effacer le cache de navigation classique de CENO (celui utilisé par le navigateur mais non partagé avec les autres), ou d'autres éléments comme les cookies, l'historique de navigation ou les favoris, vous devez aller dans les Réglages via le menu principal de l'application, puis sélectionnez Effacer les données privées. Vous serez invité à préciser les éléments que vous souhaitez effacer.

Avant de tout déposer en même temps (notamment si vous êtes pressé), apprenez à utiliser le "bouton d'urgence" décrit dans Installation de CENO.

Collecte des messages du journal

En bas de page, la case à cocher Activer le fichier journal vous permet de récupérer tous les messages internes de Ouinet et les télécharger dans un fichier. Cela ne devrait être utilisé que pour le diagnostic de problème dans CENO. Suivez simplement ces étapes :

  1. Dans la page Paramètres de CENO, cochez Activer le fichier journal.
  2. Revenez la navigation et effectuez les actions qui déclenchent le comportement inhabituel.
  3. Retournez dans la page des Paramètres de CENO et cliquez sur Télécharger à côté de la case à cocher Activer le fichier journal. Enregistrez le fichier pour une utilisation ultérieure. Android peut vous demander à ce stade si vous autorisez CENO à accéder aux médias stockés : ceci est nécessaire pour pouvoir enregistrer le fichier.
  4. Décochez la case Activer le fichier journal pour éviter que les journaux ne deviennent trop volumineux.

Vous pouvez maintenant joindre le fichier journal enregistré au rapport d'erreur. Mais évitez de le rendre public, car il peut contenir des informations sensibles sur votre navigation.
